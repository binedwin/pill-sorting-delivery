# 스마트 팩토리: 3D 비전 기반 로봇 제어 및 컨베이어 분류 자동화 시스템

**개발 환경:** Ubuntu 22.04 (ROS 2 Humble), Raspberry Pi OS, Python 3.10 / 3.11
**하드웨어 구성:** Dobot Magician, Intel RealSense d435, Raspberry Pi 5, USB 웹캠, 컨베이어 벨트(스텝모터), 분류기(서보모터)
**핵심 기술:** ROS 2 Action/Service 제어, 3D 강체 변환(Rigid Body Transformation), TCP/IP Socket 통신, OpenCV Vision AI, 비동기 스레드 모터 제어

---

## 1. 프로젝트 개요
본 프로젝트는 3D 뎁스 카메라로 물체의 정확한 물리적 위치와 높이를 파악해 로봇 팔(Dobot)이 픽업하고, 컨베이어 벨트로 이송하면 라즈베리파이가 비전 AI로 색상을 인식하여 알맞은 경로로 물리적 분류를 수행하는 End-to-End 스마트 공정 자동화 시스템입니다. 

---

## 2. 전체 시스템 파이프라인
공정은 PC(ROS 2 마스터)와 라즈베리파이(엣지 컨트롤러) 간의 유기적인 연동으로 진행됩니다.

1. **[비전 좌표 산출]** RealSense 카메라가 물체의 컬러 프레임과 뎁스(Depth) 데이터를 결합하여, 카메라 렌즈 기준의 3D 물리 좌표(mm)를 계산 후 제어 노드로 송신합니다.
2. **[로봇 픽업 및 이송]** 제어 노드가 4x4 변환 행렬을 통해 로봇 좌표로 치환하고, 물체 두께에 맞춘 동적 하강을 수행합니다. 이후 진공 흡착(Suction) 펌프를 가동해 물체를 집어 컨베이어 벨트 위에 내려놓습니다.
3. **[네트워크 트리거]** 로봇 동작이 완료되면, PC에서 라즈베리파이 측으로 개방된 포트(9090)를 통해 가동 명령(START)을 TCP/IP 소켓 통신으로 송신합니다.
4. **[컨베이어 가동]** 라즈베리파이가 신호를 수신하면 백그라운드 스레드로 스텝모터를 가동하여 상자를 웹캠 아래의 검사 위치로 이송합니다.
5. **[비전 검사 및 물리 분류]** 라즈베리파이 웹캠이 색상(Red/Blue)을 판별하고, 결과에 따라 서보모터가 지정된 각도로 움직여 차단벽을 형성합니다. 이후 컨베이어를 재가동해 상자를 최종 배출합니다.

---

## 3. 핵심 구현 기능

* **3D 공간 제어 및 동적 Z축 픽업:** 기존 2D 투시 변환의 원근 왜곡 한계를 극복하기 위해 SVD(특이값 분해) 기반의 4x4 강체 변환 행렬을 적용했습니다. 뎁스 데이터를 활용해 물체 높이에 따라 하강 깊이를 스스로 조절하는 다이나믹 픽업을 구현했습니다.
* **초경량 TCP/IP 소켓 통신 아키텍처:** 라즈베리파이에 무거운 ROS 2 생태계를 통째로 설치하는 대신, 기본 파이썬 소켓을 활용한 경량 비동기 통신망을 구축하여 시스템 부하를 줄이고 응답 속도를 극대화했습니다.
* **비동기 스레드 하드웨어 제어:** 지속적인 펄스 신호가 필요한 스텝모터 제어 로직을 Python `threading`을 활용해 백그라운드 데몬 스레드로 분리하여, 메인 비전 연산이 멈추는 블로킹 현상을 방지했습니다.

---

## 4. 트러블 슈팅 (문제 해결 과정)

### 4.1 카메라 엣지 왜곡(Edge Distortion)에 따른 로봇 오차 보정
* **문제:** 수학적 3D 변환 행렬만으로는 카메라 렌즈 고유의 물리적 어안/곡률 왜곡을 완벽히 펴지 못해, 물체가 화면 외곽에 있을 때 로봇 팔이 목표 지점을 초과하여 이동하는 현상이 발생했습니다.
* **해결:** 제어 노드 좌표 변환 로직에 스케일 보정 계수(Scale Ratio, 0.8)를 도입했습니다. 캘리브레이션 중심점을 기준으로 외곽으로 나가는 벡터의 길이를 일정 비율로 압축하도록 소프트웨어 튜닝을 거쳐 렌즈 왜곡을 상쇄했습니다.

### 4.2 뎁스 노이즈 제어 및 컬러-뎁스 정렬 불일치
* **문제:** 단일 픽셀 클릭으로 깊이를 측정 시 난반사나 플라잉 픽셀로 인해 Z축 값이 0이 되거나 튀는 현상이 잦았습니다.
* **해결:** 카메라 노드 구동 시 `align_depth` 옵션을 활성화하여 컬러 프레임과 뎁스 프레임을 1:1로 물리적 매칭시켰습니다. 또한 타겟 픽셀 단일 값이 아닌 5x5 영역의 뎁스 중앙값(Median)을 추출하는 노이즈 필터링 로직을 추가하여 충돌을 방지했습니다.

### 4.3 서보모터 미세 떨림(Jitter) 현상 제거
* **문제:** 라즈베리파이에서 OpenCV 연산과 모터 제어가 동시에 돌 때, 소프트웨어 PWM 신호 주기가 밀리면서 서보모터가 지정 각도에 안착하지 못하고 떨리는 현상이 심했습니다.
* **해결:** 모터가 회전해야 하는 약 0.4초의 시간 동안은 비전 연산을 잠시 홀딩시키고, CPU가 `lgpio`를 통한 정밀한 PWM 펄스 생성에만 집중하도록 프로세스 점유를 분리하여 떨림을 해결했습니다.

### 4.4 라즈베리파이 5 아키텍처 호환성 문제 대응
* **문제:** 라즈베리파이 5의 새로운 자체 실리콘 칩(RP1) 환경에서 기존 핀 제어 라이브러리인 `pigpio` 및 `gpiod`가 내부 아키텍처 호환성 에러를 일으켰습니다.
* **해결:** 라즈베리파이 5의 공식 로우레벨 제어 라이브러리인 `lgpio`로 전체 하드웨어 제어 코드를 전면 재작성하여 안정성을 확보했습니다.

### 4.5 카메라 버퍼 딜레이 동기화
* **문제:** 컨베이어가 상자를 2초간 이송한 직후 카메라를 켜면, 버퍼에 쌓여있던 2초 전 빈 컨베이어 프레임을 인식하여 객체 판별을 실패했습니다.
* **해결:** 비전 검사 반복문 진입 직전에 `cap.read()`를 5회 연속 호출하여 내부 버퍼에 쌓인 과거 프레임을 강제로 비워내고, 정확히 상자가 도착한 시점의 최신 프레임만 검사하도록 로직을 수정했습니다.
